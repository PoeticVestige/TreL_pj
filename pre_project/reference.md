# 悬浮阅读器软件开发规划

## 项目概述

开发一个桌面应用程序，主要功能包括：文档管理（上传、编辑、浏览）和硬件覆盖层阅读器（防录屏捕获）。

---

## 一、需求分析

### 1.1 核心功能模块

| 模块 | 功能描述 |
|------|----------|
| **文档管理模块** | 文档上传、浏览列表、简单编辑（文本内容） |
| **阅读器模块** | 硬件覆盖层窗口、防截图/录屏、快捷键控制 |
| **系统控制模块** | 快捷键监听、窗口状态管理、配置持久化 |

### 1.2 功能详细说明

**文档管理功能：**
- 支持上传常见文档格式（.txt, .md, .docx, .pdf 等）
- 文档列表展示（文件名、上传时间、文件大小）
- 简单的文本编辑功能（修改内容、保存）
- 文档删除、重命名

**阅读器功能：**
- 创建硬件加速的覆盖层窗口（绕过桌面合成器）
- 窗口始终置顶
- 显示选中的文档内容
- 支持分页显示

**快捷键控制：**
- `Ctrl + ↑/↓` 或 `Ctrl + 滚轮`：调整透明度（增减10%）
- `Ctrl + ←/→`：调整窗口宽度（增减50px）
- `Ctrl + Shift + ←/→`：调整窗口高度（增减50px）
- `PageUp/PageDown` 或 `↑/↓`：翻页
- `Esc`：隐藏/退出阅读器

## 二、技术选型

### 2.1 技术栈选择

考虑到**硬件覆盖层**和**防录屏**需求，需要访问底层图形 API：

| 方案 | 技术组合 | 优缺点 |
|------|----------|--------|
| **推荐方案A** | **Python + PyQt/PySide + Windows API** | 开发快，Python生态完善，可调用Windows DWM API创建覆盖层 |
| 方案B | C# + WPF + Windows API | 与Windows集成好，但WPF创建硬件覆盖层较复杂 |
| 方案C | Electron + Node-API + C++插件 | 界面美观，但底层调用复杂 |

**推荐采用方案A：Python + PyQt6 + pywin32**

### 2.2 依赖库

```
PyQt6 / PySide6       - GUI框架
pywin32               - Windows API调用
pymupdf / PyPDF2      - PDF解析
python-docx           - Word文档解析
keyboard              - 全局快捷键监听
pynput                - 替代键盘监听方案
```

### 2.3 硬件覆盖层实现原理

Windows 下实现防录屏窗口的关键：
- 使用 `SetWindowDisplayAffinity` API (Win10 2004+)
- 设置 `WDA_MONITOR` 标志，使窗口仅在物理显示器显示，不会被截图/录屏捕获
- 或使用 `WS_EX_LAYERED` + `SetLayeredWindowAttributes` 配合硬件加速

## 六、关键技术实现要点

### 6.1 硬件覆盖层实现（Windows）

```python
# 核心代码思路（伪代码）
import win32gui
import win32con
from ctypes import windll

# 创建窗口时设置扩展样式
ex_style = win32con.WS_EX_LAYERED | win32con.WS_EX_TOPMOST | win32con.WS_EX_TOOLWINDOW

# 设置窗口显示亲和性（防录屏核心）
WDA_MONITOR = 0x00000001
windll.user32.SetWindowDisplayAffinity(hwnd, WDA_MONITOR)
```

### 6.2 快捷键处理

```python
# 使用 keyboard 库监听全局快捷键
import keyboard

# 透明度调节
keyboard.on_hotkey('ctrl+up', increase_opacity)
keyboard.on_hotkey('ctrl+down', decrease_opacity)

# 翻页
keyboard.on_key('pageup', prev_page)
keyboard.on_key('pagedown', next_page)
```

### 6.3 分页算法

```python
def calculate_pages(text, window_width, window_height, font_metrics):
    """
    根据窗口大小和字体计算分页
    """
    chars_per_line = window_width // font_metrics.average_char_width
    lines_per_page = window_height // font_metrics.line_height
    chars_per_page = chars_per_line * lines_per_page

    # 按字符数分割文本为页
    pages = [text[i:i+chars_per_page] for i in range(0, len(text), chars_per_page)]
    return pages
```

## 八、风险与应对

| 风险 | 影响 | 应对方案 |
|------|------|----------|
| `SetWindowDisplayAffinity` 仅Win10 2004+支持 | 旧系统无法防录屏 | 检测系统版本，旧系统提示功能受限 |
| 某些录屏软件可能绕过此API | 安全性降低 | 明确告知用户这是辅助功能，非绝对安全 |
| 快捷键与其他软件冲突 | 功能失效 | 提供自定义快捷键设置 |
| 大文档加载慢 | 用户体验差 | 实现分页加载，延迟渲染 |
| PDF解析复杂 | 开发周期长 | 优先支持纯文本，PDF用pymupdf简化处理 |
